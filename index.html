Introduction to Tableau

Tableau is a leading Business Intelligence (BI) and data visualisation platform that allows users to connect, visualise, and share data insights without writing complex code. It transforms raw data into interactive, shareable dashboards.

What is Tableau?

📊
Visualisation Tool
Drag-and-drop interface to create stunning charts and graphs.
🔗
Data Connector
Connects to 70+ data sources including Excel, SQL, cloud data.
⚡
In-Memory Engine
Uses VizQL engine for fast query processing and rendering.
🌐
Collaboration
Publish & share dashboards on Tableau Server/Cloud/Public.

History & Key Milestones

Connecting to Data Sources

Tableau can connect to a vast variety of data sources — from flat files to enterprise databases and cloud services.

Types of Connections

Connection TypeExamplesNotes
File-basedExcel, CSV, JSON, PDF, XML, Spatial filesLocal files; easy drag-and-drop
Relational DBMySQL, PostgreSQL, SQL Server, OracleLive or Extract connection
Cloud ServicesGoogle BigQuery, Snowflake, AWS RedshiftFast cloud warehouses
Web ConnectorsGoogle Analytics, Salesforce, MarketoWeb Data Connector (WDC)
Statistical FilesSAS, R, SPSSStatistical analysis integration

Live vs Extract Connection

🔴
Live Connection
Queries run directly on database in real-time. Always fresh data but depends on DB speed.
🟢
Extract (.hyper)
Snapshot of data stored locally. Faster performance; can be scheduled to refresh.

Data Joins & Relationships

TypeDescription
Inner JoinReturns rows with matching values in both tables
Left JoinAll rows from left table + matching rows from right
Right JoinAll rows from right table + matching rows from left
Full Outer JoinAll rows from both tables, nulls where no match
UnionStacks rows from multiple tables with same structure
RelationshipTableau's flexible multi-table model (context-aware joins)
BlendCombines data from separate sources using a common dimension
💡

Best Practice: Use Relationships (Tableau 2020.2+) instead of joins when combining multiple tables — they preserve granularity and avoid row duplication issues.

Dimensions vs Measures

🔵
Dimensions (Blue)
Categorical, qualitative fields. Used to slice data. E.g.: Region, Category, Customer Name.
🟢
Measures (Green)
Numerical, quantitative fields. Aggregated by default. E.g.: Sales, Profit, Quantity.

Discrete vs Continuous

🔵
Discrete (Blue Pill)
Individual, distinct values. Creates column/row headers. E.g.: Month names, Category.
🟢
Continuous (Green Pill)
Unbroken range of values. Creates axes. E.g.: Continuous date axis, Sales range.

Aggregation Functions

Filters in Tableau

Filters allow you to control which data is displayed. Tableau applies filters in a specific Order of Operations.

Order of Operations (Filter Hierarchy)

⬇ Filter Execution Order (Top = Applied First)

1. Extract Filters  →  2. Data Source Filters  →  3. Context Filters  →  4. Top N / Condition Filters  →  5. Dimension Filters  →  6. Measure Filters  →  7. Table Calculation Filters

Filter Types

Filter TypeDescriptionUse Case
Extract FilterApplied when creating the .hyper extractReduce extract size
Data Source FilterApplied to the data source; shared across sheetsRow-level security
Context FilterCreates a temporary table for dependent filtersTop N filtered from subset
Dimension FilterFilters on discrete/categorical valuesShow only Region = East
Measure FilterFilters aggregated numeric valuesShow SUM(Sales) > 10,000
Table Calc FilterFilters after table calculations completeFilter on rank or running total

Quick Filters & Filter Controls

Single Value (List) Multiple Values (List) Single Value (Dropdown) Slider Wildcard Match Relative Date Date Range
💡

Apply to Worksheets: Right-click a filter → "Apply to Worksheets" → choose "All Using This Data Source" to share filters across sheets in a dashboard.

Calculated Fields & LOD Expressions

Calculated Fields

Created via Analysis → Create Calculated Field. Written using Tableau's formula language with functions similar to Excel.

Common Function Categories

CategoryExample Functions
StringLEFT(), RIGHT(), MID(), LEN(), CONTAINS(), REPLACE(), TRIM()
NumberROUND(), ABS(), CEILING(), FLOOR(), POWER(), SQRT(), ZN()
DateDATEPART(), DATENAME(), DATEADD(), DATEDIFF(), TODAY(), NOW()
LogicalIF, IIF, CASE, IFNULL, ISNULL, ZN, AND, OR, NOT
AggregateSUM(), AVG(), COUNT(), COUNTD(), MIN(), MAX(), MEDIAN()
Type ConversionSTR(), INT(), FLOAT(), DATE(), DATETIME()

Sample Calculated Fields

// Profit Margin
SUM([Profit]) / SUM([Sales])

// IF-THEN-ELSE
IF [Sales] > 10000 THEN "High"
ELSEIF [Sales] > 5000 THEN "Medium"
ELSE "Low"
END

// CASE statement
CASE [Region]
  WHEN "East" THEN "Eastern"
  WHEN "West" THEN "Western"
  ELSE "Other"
END

Level of Detail (LOD) Expressions

LOD expressions let you control the granularity of aggregation independently of the view.

LOD TypeSyntaxDescription
FIXED{FIXED [Dim] : AGG([Measure])}Computes at specified dimension level, ignores view filters
INCLUDE{INCLUDE [Dim] : AGG([Measure])}Adds dimension to current view level
EXCLUDE{EXCLUDE [Dim] : AGG([Measure])}Removes dimension from current view level
// Customer's first purchase date (FIXED LOD)
{FIXED [Customer ID] : MIN([Order Date])}

// Sales per city regardless of view (FIXED LOD)
{FIXED [City] : SUM([Sales])}

// Average sales including sub-category (INCLUDE LOD)
{INCLUDE [Sub-Category] : AVG([Sales])}

CalculationFunctionUse Case

Parameters

Parameters are dynamic, user-controlled variables that can replace constant values in calculations, filters, and reference lines.

Creating a Parameter

1
Right-click in the Data pane → Create Parameter
2
Set Name, Data Type (Integer, Float, String, Boolean, Date)
3
Set Allowable values: All, List, or Range
4
Right-click parameter → Show Parameter Control to expose in view
5
Reference in a calculated field: e.g., SUM([Sales]) > [Min Sales Parameter]

Common Parameter Use Cases

🔢
Top N Filter
Let users choose how many top items to display.
📏
Reference Line Value
Dynamic target or benchmark line controlled by user.
🔀
Measure Switcher
Switch between Sales, Profit, Quantity on a single chart.
📅
Date Granularity
Switch between Day / Month / Quarter / Year views.

Tableau Prep & Data Cleaning

Tableau Prep Builder is a visual, drag-and-drop data preparation tool for cleaning, shaping, and combining data before analysis in Tableau Desktop.

Key Tableau Prep Steps

📥
Input Step
Connect to data sources (files, databases, cloud).
🧹
Clean Step
Remove nulls, fix data types, rename fields, filter rows.
🔗
Join Step
Combine tables using joins (inner, left, right, full).
📚
Union Step
Stack rows from multiple tables with same structure.
📊
Aggregate Step
Group data and compute aggregated metrics.
🔄
Pivot Step
Pivot columns to rows or rows to columns (reshape).
📤
Output Step
Save as .hyper extract, CSV, or publish to Tableau Server.

Row-Level Security (RLS)

Restrict data access per user. Use User Functions like USERNAME() or USERDOMAIN() in calculated fields and data source filters to filter data based on who is viewing the dashboard.

// RLS Calculated Field Example
[Region] = LOOKUP(USERNAME(), [User-Region Mapping])

Scheduling Extracts & Subscriptions

Performance & Best Practices

Performance Tips

AreaBest Practice
Data SourceUse Extracts (.hyper) instead of Live for large datasets
Data SourcePush aggregation to database with custom SQL or views
CalculationsPrefer FIXED LOD over complex joins; avoid nested LODs
CalculationsUse native aggregations instead of row-level calculations
FiltersUse Context Filters for Top N; apply Data Source Filters early
DashboardsLimit to 3–5 sheets per dashboard; avoid too many filters
DashboardsUse worksheet actions instead of quick filters for interactivity
MarksKeep mark count under 100K for rendering performance
MapsUse Polygon maps sparingly; prefer symbol maps for large data

Design Best Practices

🎯
Start with a Question
Define the business question before building any chart.
🎨
Colour Wisely
Use colour for meaning (sequential, diverging, categorical), not decoration.
✂️
Reduce Clutter
Remove gridlines, borders, and tooltips that don't add value.
📐
Consistent Formatting
Use one font family, aligned axis labels, and consistent date formats.

Performance Recording

Use Help → Settings and Performance → Start Performance Recording to identify slow queries, layout rendering, and data computation bottlenecks.

🚀 Remember – The Golden Rule of Tableau

Always ask: "What question does this visualisation answer?" Every design choice — chart type, colour, filter, layout — should serve that question. Data tells a story; your job is to make it clear.
